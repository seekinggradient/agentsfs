01

Real Git remote

Push from where the work happens.

Connect the CLI once, then send any AgentsFS knowledge base to your private namespace. Every sync uses ordinary Git to push the commits your agents already made.

$ afs hub login$ afs hub push insurance-claim
02

Web workspace

Browse and edit the knowledge base.

Browse a described file tree, follow wikilinks and backlinks, edit notes, see the knowledge graph, and inspect the Git history.

hub.agentsfs.ai/demo/insurance-claim↗
03

Complete checkout

Pull the repository back at any time.

afs hub pull downloads the files and complete Git history. Keep working locally, add another remote, or run the Hub on infrastructure you control.

$ afs hub pull insurance-claim$ git log --oneline

Portable by construction

The Hub never becomes a new file format.

Local changes and web edits are ordinary Git commits. Pull before editing, resolve divergent histories with normal Git tools, and move to another remote without converting your data.

download a complete checkout
$ afs hub pull insurance-claim
